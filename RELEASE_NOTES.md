# RELEASE NOTES

## Версия 0.1.0

Дата релиза: `2026-02-28`

## Что вошло в релиз

Версия `0.1.0` фиксирует первый цельный контур проекта `202312-todo`: локальный ToDo-менеджер на Python с SQLite, рабочим CLI и минимальным REST API на FastAPI.

В релиз вошли:

- CLI для основных операций с задачами;
- локальная SQLite-база без внешних зависимостей на сервер БД;
- функции создания, просмотра, завершения и удаления задач;
- поддержка deadline;
- минимальный HTTP API для чтения задач;
- конфигурация через `.ini`;
- логирование CLI;
- тесты и примерные данные для локальной проверки.

## Почему это важно

Этот релиз ценен не широтой возможностей, а тем, что проект уже решает практическую задачу без лишней инфраструктуры. Для локального task tracking достаточно:

- одного репозитория;
- одного файла SQLite;
- стандартного Python-окружения;
- простого CLI.

Это хороший базовый слой для личного использования, экспериментов с FastAPI и дальнейшего развития в сторону более формального task manager.

## Практическая ценность

Релиз полезен в нескольких реальных сценариях:

- вести локальный список задач без web UI и облачной зависимости;
- быстро проверять CRUD-логику на Python + SQLite;
- использовать проект как основу для легкого REST API;
- развивать собственный CLI-инструмент для personal productivity.

## Технические акценты релиза

- Основной рабочий путь сейчас проходит через CLI.
- API уже можно использовать для чтения данных по HTTP.
- Модель данных предельно простая и понятная: одна таблица с полями задачи, датой создания, deadline и признаком выполнения.
- Проект остается компактным и пригодным для локальной отладки.

## Ограничения релиза

Релиз `0.1.0` не стоит трактовать как production-ready выпуск.

Текущие ограничения:

- API ограничен и не выглядит стабилизированным контрактом.
- Формат дат строковый и ориентирован в первую очередь на CLI.
- Некоторые CLI-операции интерактивны и требуют подтверждения.
- Архитектура пока ближе к практическому MVP, чем к формально выстроенному приложению.
- Часть тестовой воспроизводимости зависит от локальной среды, в частности от `uv`.

## Что логично делать дальше

Наиболее разумные следующие шаги:

1. Стабилизировать API и добавить операции записи.
2. Упростить автоматизированный запуск CLI без интерактивных подтверждений.
3. Улучшить формализацию схемы данных и работы с датами.
4. Расширить тестовый контур так, чтобы он меньше зависел от окружения.
5. При необходимости добавить web UI или внешний клиент.
